# 昭和像素风视觉规范

这份规范先用来约束下一轮美术重做，避免继续出现“功能能跑但画风突兀”的问题。

## 关键词

- 昭和日式小家
- 像素绘制感
- 木质、旧纸、暖灯、生活感
- 精致但不现代扁平

## 色彩

- 主背景：旧纸米色、奶油色、浅桃杏色。
- 木结构：低饱和棕色、焦糖木色、深茶色描边。
- 强调色：暖橙、樱粉、紫阳花紫、复古金。
- 夜晚：房间整体压暗，但灯光照到的区域必须真实变亮。
- 雨天：只影响窗外/庭院，室内不出现整屏雨线。
- 雪天：窗外与庭院变冷白浅蓝，室内只受外部冷光轻微影响。

## 像素资产

- 家具、图标、窗口边框必须像同一套资产，避免 CSS 扁平图形和精细像素图混用。
- 图标优先画成具体物件：存钱罐、信箱、日记本、相册、电脑，而不是抽象方块。
- 每个资产需要保留 1-2 像素深色描边、局部高光和投影。
- 家具在房间里必须吃同一套光线：白天偏亮、夜晚压暗、灯打开后受暖光影响。

## OS 桌面

- 窗口边框要像旧式电脑窗口，但材质偏和纸/木边框，不要现代玻璃感。
- 标题栏可以用浅桃杏渐变、深木描边、像素按钮。
- 按钮需要有明确的凹凸层次，不能只是 CSS 方框加文字。
- 桌面图标使用 64-96px 像素小物件，文字只作为标签。

## 家园和房间

- 家园先不继续堆功能，优先统一美术：底图、动态层、可摆放家具的透视和光照。
- 室内植物默认不动；风、雨、雪主要发生在庭院或窗外。
- 灯具必须可单独控制：顶灯、小台灯、庭院灯互不联动。
- 互动家具应当有状态：电脑开合、日记本开合、灯开关、信箱有无新信。

## 下一轮改造顺序

1. 先重画 OS 窗口边框和桌面图标。
2. 再统一家具光照和比例。
3. 然后整理家园/房间底图和动态层。
4. 最后再接真实 AI、记忆库和后端数据。
